# 狼人杀笔记助手

狼人杀笔记助手是一个桌面优先的第二屏 Web App，适合玩家一边在手机上玩狼人杀，一边在电脑上快速记录结构化对局信息。

它的目标不是写长篇笔记，而是尽量用点击完成记录：

- 玩家身份倾向
- 跳预 / 金水 / 查杀 / 站边 / 打 / 保
- 警上 / 警下发言
- 投票关系
- 出局信息
- 阶段推进（第1天、第1夜、第2天……）

## 技术栈

- Next.js（App Router）
- React
- Tailwind CSS
- Zustand
- LocalStorage
- Vitest + Testing Library

## 环境要求

建议使用以下环境：

- Node.js 20 或更高版本
- npm 10 或更高版本

如果你的电脑还没有安装 Node.js，可以先去官网下载并安装：

- [Node.js 官网](https://nodejs.org/)

安装完成后，打开终端执行下面命令确认：

```bash
node -v
npm -v
```

## 第一次使用

1. 打开首页。
2. 点击 `新建对局`。
3. 选择玩家人数：`9 / 10 / 12`。
4. 点击 `开始记录`，进入对局页面。

## 对局页面怎么用

### 1. 左侧玩家面板

左侧会显示所有玩家卡片。

你可以直接点击或切换：

- `存活 / 已出局`
- 身份倾向：`未知 / 好人 / 狼人 / 神职`
- 标签：`跳预 / 警长 / 对跳 / 金水 / 查杀`
- `盯`：把玩家加入重点怀疑对象

点击玩家卡片后，会激活这个玩家并打开中间的快捷记录区。

### 2. 中间阶段区

顶部是阶段标签页，例如：

- `第1天`
- `第1夜`
- `第2天`

点击 `进入下一阶段` 会自动追加下一阶段，并切换过去。

白天阶段包含：

- 警上发言
- 警下发言
- 投票关系
- 出局信息

夜晚阶段包含：

- 夜间记录

### 3. 快捷记录

选中玩家后，在中间的 `快捷操作` 区域可以快速记录：

- 跳预
- 金水
- 查杀
- 站边
- 打
- 保

如果需要目标玩家，可以继续点选目标玩家号码。

例如：

- `9号：查杀5号`
- `4号：站边2号`

点击 `记录到当前阶段` 后，会自动插入到当前记录分区。

### 4. 白天记录分区

你可以先选择当前写入位置：

- `警上发言`
- `警下发言`

这样后续所有快捷记录都会进入该分区，直到你手动切换。

### 5. 投票关系

在投票区依次点击：

1. 投票人
2. 投票目标
3. `保存投票`

系统会显示投票列表，并自动计算当前最高票：

- 如果有唯一最高票，会显示当前领先玩家
- 如果平票，会显示 `平票`
- 如果没有记录，会显示 `暂无记录`

### 6. 出局信息

在出局区可以：

- 选择出局玩家
- 给该玩家设置真实身份：`未知 / 好人 / 狼人 / 神职`

设置后，右侧摘要区会同步更新。

### 7. 右侧摘要区

右侧会实时显示：

- 当前警长
- 存活人数
- 当前阶段
- 出局名单
- 重点怀疑对象

## 自动保存说明

本项目没有后端，所有数据都保存在浏览器本地的 LocalStorage 里。

这意味着：

- 刷新页面后，对局通常会自动恢复
- 同一个浏览器里会保留当前对局
- 点击 `新建对局` 会覆盖当前自动保存记录

当前 MVP 只保留一局“当前对局”，不支持多存档。

## 项目结构

主要文件如下：

```text
app/
  page.tsx                首页
  game/page.tsx           对局页
components/
  home/home-page.tsx      首页界面

  game/helpers.ts         阶段与显示辅助函数

  *.test.ts(x)            基础测试
```

## 注意事项

- 这是桌面优先界面，推荐在电脑浏览器中使用
- 所有数据都在本地浏览器中，清除浏览器站点数据后记录也会消失
- 如果你换浏览器或换电脑，当前记录不会自动同步
